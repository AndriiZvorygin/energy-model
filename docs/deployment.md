# Website Deployment

The educational website is a static Vite application under `website/`. It reads generated research artifacts during the build and does not require a Python server in production.

Before deployment, verify that the intended files under `analysis/` and `charts/` have been generated by the Python pipeline. Then install and build the website:

```bash
cd website
npm install
npm run build
```

The deployable static files are written to `website/dist/`.

## Option A: GitHub Pages

For the repository `AndriiZvorygin/energy-model`, build with the repository base path:

```bash
cd website
npm install
npm run build:pages
```

Publish the contents of `website/dist/` with a GitHub Actions Pages workflow or a dedicated deployment branch. The Pages build also creates `dist/404.html` so direct requests to client-side routes return the application shell. In the repository settings, choose **GitHub Actions** as the Pages source.

A typical workflow should:

1. check out the repository;
2. set up Node.js;
3. run `npm ci` inside `website/`;
4. run `npm run build:pages`;
5. upload `website/dist/` as the Pages artifact;
6. deploy the artifact with the official GitHub Pages action.

The resulting project URL is normally:

```text
https://andriizvorygin.github.io/energy-model/
```

The site uses the Vite base path for chart assets and React Router, so navigation within the deployed application remains under `/energy-model/`.

## Option B: Custom Domain

The proposed custom-domain path is:

```text
https://andrii.zvorygin.ca/energy-model/
```

Build with the same subpath base:

```bash
cd website
npm install
npm run build -- --base /energy-model/
```

Serve `website/dist/` from the `/energy-model/` path and configure the web server to return `index.html` for client-side routes such as `/energy-model/liquidity` and `/energy-model/economy`.

If the site later moves to the root of a dedicated domain, use the standard `npm run build` command instead. That build uses `/` as its base path.

## Updating Published Research

When the Python analysis changes intentionally:

```bash
.venv/bin/python -m oil_model.pipeline --root .
.venv/bin/python -m oil_model.verify_release --root .
cd website
npm run build
```

The website's prebuild step regenerates `src/data/generated.ts` from selected CSV and Markdown files and copies selected PNGs into `website/public/charts/`. Review those generated presentation assets before deployment.
